# Run a full node on Eden mainnet

This guide walks through running an Eden full node using Docker Compose. The stack consists of two
services:

- **ev-reth** — the EVM execution layer (handles RPC, transaction processing, and state)
- **ev-node** — the consensus/DA layer (fetches blobs from Celestia and drives ev-reth via the
  Engine API)

## Prerequisites

- Docker and Docker Compose v2 installed
- A running [Celestia light node](https://docs.celestia.org/nodes/light-node) connected to
  Celestia mainnet, with its RPC accessible from this machine

## Download the artifacts

Download the Eden mainnet deployment package. It contains the `compose.yaml`, genesis files, and
helper scripts:

```bash
# Clone or download from the Eden repository
git clone TODO: setup repo for artifacts
```

The directory should contain:

```
compose.yaml
ev-node.genesis.json
ev-reth.genesis.json
entrypoint.fullnode.sh
init-1-ev-node.sh
init-1-ev-reth-snapshot.sh
init-2-ev-node-snapshot.sh
```

## Configure the environment

Create a `.env` file in the same directory as `compose.yaml`:

```bash
# .env

# --- Required ---
# Address of your Celestia light node RPC
DA_ADDRESS=http://your-celestia-light-node:26658

# --- Port configuration ---
EV_RETH_ADDRESS=127.0.0.1
EV_RETH_RPC_PORT=8545
EV_RETH_WS_PORT=8546
EV_RETH_ENGINE_PORT=8551
EV_RETH_DISCOVERY_PORT=30313
EV_RETH_PROMETHEUS_PORT=9001

EV_NODE_RPC_PORT=7676
EV_NODE_PROMETHEUS_PORT=26660
```

The DA namespace variables (`DA_HEADER_NAMESPACE`, `DA_DATA_NAMESPACE`) are mainnet-specific values
that are pre-set in the distributed package; do not change them unless you know what you are doing.

## Choose a sync method

:::tip[Recommended]
**Snapshot restore** is much faster — the node is operational within minutes instead of syncing
from block 0.
:::

### Method A: Restore from snapshot (recommended)

The default `compose.yaml` includes init containers that automatically download the latest database
snapshots from Binary Builders storage before starting ev-reth and ev-node. No changes to
`compose.yaml` are needed.

Start the stack:

```bash
docker compose up -d
```

Monitor the snapshot download progress, then watch ev-node sync:

```bash
docker compose logs -f init-1-snapshot-ev-reth
docker compose logs -f init-2-snapshot-ev-node
docker compose logs -f ev-node
```

### Method B: Sync from genesis

This method starts from block 0. Sync time depends on your connection and hardware.

Edit `compose.yaml` to remove the snapshot init containers and their dependencies:

1. Delete the `init-1-snapshot-ev-reth` and `init-2-snapshot-ev-node` service blocks.

2. In the `ev-reth` service, update `depends_on` to remove `init-1-snapshot-ev-reth`:

   ```yaml
   ev-reth:
     depends_on:
       jwt-init:
         condition: service_completed_successfully
   ```

3. In the `ev-node` service, update `depends_on` to remove `init-2-snapshot-ev-node`:

   ```yaml
   ev-node:
     depends_on:
       ev-reth:
         condition: service_started
   ```

Then start the stack:

```bash
docker compose up -d
docker compose logs -f ev-node
```

## Verify the node is running

Check that ev-reth is responding to RPC calls:

```bash
curl -s -X POST -H "Content-Type: application/json" \
  --data '{"jsonrpc":"2.0","method":"eth_blockNumber","params":[],"id":1}' \
  http://localhost:8545
```

Check ev-node health:

```bash
curl -s http://localhost:7676/health/live
```

## Monitor sync progress

```bash
# Follow all service logs
docker compose logs -f

# Follow a specific service
docker compose logs -f ev-node
docker compose logs -f ev-reth
```

Prometheus metrics are available at:

- ev-reth: `http://localhost:9001`
- ev-node: `http://localhost:26660`
