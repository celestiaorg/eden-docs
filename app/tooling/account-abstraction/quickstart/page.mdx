# Account abstraction with ZeroDev & Privy

import { Callout, Steps } from 'nextra/components'

<Callout type="info" emoji="🚧">
  This page is a work in progress. It currently uses Ethereum Sepolia testnet.
</Callout>

Account Abstraction (AA) transforms user accounts into smart contracts, unlocking powerful features that go beyond traditional Externally Owned Accounts (EOAs). With AA, you can offer users a Web2-like experience, including social logins, gasless transactions, and enhanced security, without them ever needing to manage seed phrases.

This guide will walk you through building a simple Next.js application on Eden that uses [Privy](https://privy.io) for email-based social logins and [ZeroDev](https://zerodev.app) to sponsor gas fees, allowing users to mint an NFT for free.

<Callout type="info">
  This guide is based on the [`zerodev-privy-demo`](https://github.com/gbarros/zerodev-privy-demo)
  project in this repository. You can explore the full source code for a complete reference.
</Callout>

## Prerequisites

Before you begin, make sure you have the following:

- **Node.js**: Ensure you have a recent version of Node.js (v18+) installed.
- **Privy Account**: Sign up for a free account at [Privy](https://privy.io) and create a new app. You will need the **App ID**.
- **ZeroDev Account**: Sign up at [ZeroDev](https://zerodev.app) and create a new **v3 Project**. You will need the **Project ID**, which gives you access to the Bundler and Paymaster RPC URLs.
- **NFT Contract**: A deployed NFT smart contract on the Eden testnet. The contract should have a public `mint()` or `safeMint(address to)` function. You will need its **contract address**.

## Dashboard setup

Before writing code, you need to configure your Privy and ZeroDev accounts properly.

### Privy dashboard configuration

<Steps>

#### Create a new app

1. Go to the [Privy Dashboard](https://dashboard.privy.io)
2. Create a development account with your email
3. Click **"+ New app"**
4. Give your app a name, select **"Development"** environment, and choose your platforms
5. Save your App Secret and copy your **App ID** - you'll need this for your `.env.local` file
6. Follow their "Get started" guide to configure your app (default options work for this tutorial)

#### Configure login methods and enable embedded wallets

1. In your app dashboard, go to **"Configure user login methods"**
2. Enable **"Automatically create embedded wallets on login"**
3. Disable other login methods if you only want email authentication
4. Save your changes

</Steps>

### ZeroDev dashboard configuration

<Steps>

#### Create a new project

1. Go to the [ZeroDev Dashboard](https://dashboard.zerodev.app)
2. Sign up or log in with your existing account
3. Click **"Create Project"**
4. Choose **"Kernel v3"** (latest version)
5. Select **"Celestia testnet"** as the network (chain ID "3735928814")

#### Configure gas policies

1. In your project dashboard, go to **"Gas Policies"**
2. Click **"Create Policy"**
3. Set up a policy that allows:
   - Contract calls to your NFT contract address
   - Gas sponsorship for `mint` function calls
4. Ensure there are no errors when saving the policy

<Callout type="info">
  For development, selecting "Sponsor all transactions" is the recommended policy for simplicity.
</Callout>

#### Create and fund your paymaster

1. In your project dashboard, go to **"Self Funded Paymasters"**
2. Connect your wallet
3. Under "Verifying Paymaster", click **"Deploy"** to create your paymaster
4. Copy the paymaster **RPC URL** from the dashboard - you'll need it in your `.env.local` file
5. Fund your paymaster with some Eden testnet tokens

#### Get your RPC URLs

1. Go to **"General"** in your project dashboard
2. Copy the **Bundler RPC URL** (e.g., `https://rpc.zerodev.app/api/v3/%AppId%/chain/3735928814`)
3. Copy the **Paymaster RPC URL** (from step 3.4 above)
4. These URLs will be used in your `.env.local` file

<Callout type="warning">
  Keep your RPC URLs secure and never commit them to public repositories. They provide access to
  your gas sponsorship budget.
</Callout>

</Steps>

## Step-by-step guide

<Steps>

### Set up your Next.js project

First, create a new Next.js application and install the required dependencies.

```bash filename="Terminal"
npx create-next-app@latest eden-aa-demo --typescript --tailwind --eslint --app --src-dir
cd eden-aa-demo
npm install @privy-io/react-auth @zerodev/sdk @zerodev/ecdsa-validator viem
```

This sets up a new Next.js project with TypeScript and Tailwind CSS and installs the necessary libraries for authentication and account abstraction.

### Configure environment variables

Create a `.env.local` file in the root of your project to store your secret keys and project IDs.

```env filename=".env.local"
# Get this from your Privy dashboard
NEXT_PUBLIC_PRIVY_APP_ID=YOUR_PRIVY_APP_ID

# Get these from your ZeroDev v3 Project dashboard
NEXT_PUBLIC_ZERODEV_BUNDLER_RPC=YOUR_ZERODEV_BUNDLER_RPC_URL
NEXT_PUBLIC_ZERODEV_PAYMASTER_RPC=YOUR_ZERODEV_PAYMASTER_RPC_URL

# The address of your deployed NFT contract on Eden testnet
NEXT_PUBLIC_NFT_CONTRACT_ADDRESS=YOUR_NFT_CONTRACT_ADDRESS
```

<Callout type="warning">
  Remember to replace the placeholder values with your actual credentials from the Privy and ZeroDev
  dashboards.
</Callout>

### Set up the Privy Provider

Privy provides a React hook to manage user authentication. Wrap your application with `PrivyProvider` to make these hooks available everywhere.

Create a new file at `src/app/providers.tsx`:

```tsx filename="src/app/providers.tsx"
'use client'

import { PrivyProvider } from '@privy-io/react-auth'
import React from 'react'

export default function Providers({ children }: { children: React.ReactNode }) {
  return (
    <PrivyProvider
      appId={process.env.NEXT_PUBLIC_PRIVY_APP_ID!}
      config={{
        loginMethods: ['email'],
        appearance: {
          theme: 'light',
          accentColor: '#676FFF'
        },
        embeddedWallets: {
          createOnLogin: 'users-without-wallets'
        }
      }}
    >
      {children}
    </PrivyProvider>
  )
}
```

Now, apply this provider in your root layout file, `src/app/layout.tsx`:

```tsx filename="src/app/layout.tsx"
import './globals.css'
import Providers from './providers'

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        <Providers>{children}</Providers>
      </body>
    </html>
  )
}
```

### Build the minting component

This is where the core logic resides. Create a component at `src/components/MagicMint.tsx`. This component will handle:

- Logging the user in via Privy.
- Creating a ZeroDev smart account client using the user's Privy wallet as a signer.
- Sending a gas-sponsored transaction to mint an NFT.

Here is the full implementation:

```tsx filename="src/components/MagicMint.tsx"
'use client'

import { useCallback, useEffect, useState } from 'react'
import { usePrivy, useWallets } from '@privy-io/react-auth'
import {
  createKernelAccount,
  createKernelAccountClient,
  createZeroDevPaymasterClient,
  constants
} from '@zerodev/sdk'
import { signerToEcdsaValidator } from '@zerodev/ecdsa-validator'
import { http, createPublicClient, EIP1193Provider } from 'viem'
import { sepolia } from 'viem/chains' // Replace with Eden chain if available
import { nftContractAbi, nftContractAddress } from '@/lib/contract' // We'll create this next

export default function MagicMint() {
  const { login, ready, authenticated, logout } = usePrivy()
  const { wallets } = useWallets()
  const [kernelClient, setKernelClient] = useState<any>(null)
  const [saAddress, setSaAddress] = useState<string | null>(null)
  const [txHash, setTxHash] = useState<string | null>(null)
  const [loading, setLoading] = useState(false)

  const setupSmartAccount = useCallback(async () => {
    const embeddedWallet = wallets.find(w => w.walletClientType === 'privy')
    if (!embeddedWallet) return

    const provider = (await embeddedWallet.getEthereumProvider()) as EIP1193Provider
    const publicClient = createPublicClient({ chain: sepolia, transport: http() })

    // Create ZeroDev validator & account
    const ecdsaValidator = await signerToEcdsaValidator(publicClient, {
      signer: provider,
      entryPoint: constants.getEntryPoint('0.7'),
      kernelVersion: constants.KERNEL_V3_3
    })

    const account = await createKernelAccount(publicClient, {
      plugins: { sudo: ecdsaValidator },
      entryPoint: constants.getEntryPoint('0.7'),
      kernelVersion: constants.KERNEL_V3_3
    })

    // Configure ZeroDev Paymaster client and wire into Kernel client
    const paymaster = createZeroDevPaymasterClient({
      chain: sepolia,
      transport: http(process.env.NEXT_PUBLIC_ZERODEV_PAYMASTER_RPC!)
    })

    const client = createKernelAccountClient({
      account,
      chain: sepolia,
      bundlerTransport: http(process.env.NEXT_PUBLIC_ZERODEV_BUNDLER_RPC!),
      paymaster
    })

    setKernelClient(client)
    setSaAddress(client.account.address)
  }, [wallets])

  useEffect(() => {
    if (authenticated && wallets.length > 0) {
      setupSmartAccount()
    }
  }, [authenticated, wallets, setupSmartAccount])

  const handleMint = async () => {
    setLoading(true)
    const hash = await kernelClient.writeContract({
      address: nftContractAddress,
      abi: nftContractAbi,
      functionName: 'mint',
      args: []
    })
    setTxHash(hash)
    setLoading(false)
  }

  if (!ready) return <p>Loading...</p>

  if (!authenticated) {
    return <button onClick={login}>Log In</button>
  }

  return (
    <div>
      <button onClick={logout}>Log Out</button>
      {saAddress && <p>Smart Account: {saAddress}</p>}
      <button onClick={handleMint} disabled={loading || !saAddress}>
        {loading ? 'Minting...' : 'Claim Free NFT'}
      </button>
      {txHash && (
        <a href={`https://sepolia.etherscan.io/tx/${txHash}`} target="_blank">
          View Transaction
        </a>
      )}
    </div>
  )
}
```

<Callout>
  The code above uses `sepolia` from `viem/chains`. You should replace this with the configuration
  for **Eden testnet**.
</Callout>

### Configure contract and finalize UI

Create a file at `src/lib/contract.ts` to hold your NFT contract's address and ABI.

```ts filename="src/lib/contract.ts"
export const nftContractAddress = process.env.NEXT_PUBLIC_NFT_CONTRACT_ADDRESS as `0x${string}`

export const nftContractAbi = [
  {
    type: 'function',
    name: 'mint',
    stateMutability: 'nonpayable',
    inputs: [],
    outputs: []
  }
] as const
```

Finally, update your main page at `src/app/page.tsx` to render the component.

```tsx filename="src/app/page.tsx"
import MagicMint from '@/components/MagicMint'

export default function Home() {
  return (
    <main className="flex min-h-screen flex-col items-center justify-center p-24">
      <h1 className="text-4xl font-bold mb-8">Account Abstraction Demo</h1>
      <MagicMint />
    </main>
  )
}
```

</Steps>
