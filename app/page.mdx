# About


## Native TIA DeFi. Powered by Celestia underneath.

**Celestia Eden** is a new high-performance EVM network built using the [Evolve stack](https://github.com/evstack) and powered by **Celestia for data availability**.

It allows DeFi builders to launch composable apps that natively tap into the TIA ecosystem— with deep liquidity and without reinventing their infra stack.


## Why another network?

Right now, there's no meaningful DeFi ecosystem built specifically for Celestia. **Eden fills that gap**:

- A ready-to-deploy EVM chain, optimized for high throughput
- Built with Evolve Stack
- Native liquidity layer to the TIA token, designed for composable DeFi

Built to showcase what TIA is capable of.


## Who is Celestia Eden for?

**Eden is built for app developers and DeFi users.**

Whether you're:

- A DeFi team exploring Celestia-native deployments
- Porting your app from another EVM chain

**Eden gives you a clean, scalable canvas to launch.**

## The Stack Behind Eden

Celestia Eden is powered by the **Evolve stack**:
| Component | Description |
| --- | --- |
| `ev-node` | Modular node engine connecting execution to Celestia DA |
| `ev-reth` | High-performance EVM implementation |
| `ev-toolbox` | Tools for testing and deployment |

Built with modularity at its core. Powered by Celestia's DA layer.



## FAQs

**Is Eden its own network?**

> Yes. Eden is a standalone EVM chain that uses Celestia for DA.

**Is this part of Celestia mainnet?**

> Eden posts data to Celestia mainnet, but runs its own execution environment. Celestia only handles consensus and data availability.

**Do I need testnet tokens?**

> Yes. You can get them from the faucet. Join the Telegram to request access.

**Can I launch my own network using Eden?**

> No—Eden is for apps, not networks. But if you want to launch your own chain, check out the Evolve stack.

**How is this different from other EVM chains?**

> Eden uses a modular stack with no validator overhead and is natively tied to Celestia's DA layer, giving you performance without the emissions burden.
