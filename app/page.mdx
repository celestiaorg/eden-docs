Celestia Eden is an EVM network built using the [Evolve stack](https://github.com/evstack) and powered by Celestia for data availability.

## Overview

- EVM chain optimized for high throughput
- Built with Evolve Stack
- Built for EVM applications on Celestia
- Uses Celestia for data availability

## Target users

- EVM builders of all stripes (game, consumer, social, etc.)
- Teams porting apps from other EVM chains

## Features

- 🧱 EVM-ready environment (built with ev-reth)
- ⚙️ Single sequencer for fast, predictable block production
- 🔗 Native TIA deposits through the Eden bridge
- 🔭 Explorer access to track your contracts
- 🧪 Production-grade infra for mainnet applications

## The stack behind Eden

Celestia Eden is powered by the **Evolve stack**:
| Component | Description |
| --- | --- |
| `ev-node` | Modular node engine connecting execution to Celestia DA |
| `ev-reth` | High-performance EVM implementation |
| `ev-toolbox` | Tools for testing and deployment |

## FAQs

<details>
  {/* prettier-ignore */}
  <summary>Is Eden its own network?</summary>
  Yes. Eden is a standalone EVM chain that uses Celestia for DA.
</details>

<details>
  {/* prettier-ignore */}
  <summary>Is this part of Celestia mainnet?</summary>
  Eden posts data to Celestia mainnet, but runs its own execution environment. Celestia only handles
  consensus and data availability.
</details>

<details>
  {/* prettier-ignore */}
  <summary>Do I need TIA?</summary>
  Yes. Eden uses TIA as its native gas token. Use the Eden bridge to deposit TIA before sending
  transactions.
</details>

<details>
  {/* prettier-ignore */}
  <summary>Can I launch my own network using Eden?</summary>
  No — Eden is for apps, not networks. But if you want to launch your own chain, check out the
  Evolve stack.
</details>

<details>
  {/* prettier-ignore */}
  <summary>How is this different from other EVM chains?</summary>
  Eden uses a modular stack with no validator overhead and is natively tied to Celestia's DA layer,
  giving you performance without the emissions burden.
</details>
