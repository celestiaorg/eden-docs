# Eden / Foundry Quickstart

import { EDEN_NETWORKS } from '../../../constants'
import { Callout } from 'nextra/components'

A minimal guide to deploying and interacting with smart contracts on **Celestia Eden** using [Foundry](https://book.getfoundry.sh/).


## Prerequisites

* [Foundry](https://book.getfoundry.sh/getting-started/installation) installed (`foundryup`)
* A funded Eden testnet address (use the [Eden faucet](/networks/faucet))


## 1. Initialize a project

```bash
forge init eden-forge && cd eden-forge
```

## 2. Configure `foundry.toml`

```toml
[profile.default]
src = "src"
out = "out"
libs = ["lib"]
optimizer = true
optimizer_runs = 200

[rpc_endpoints]
eden = "http://rpc-evnode-edennet-1-testnet.binary.builders:8080"
```

<Callout type="info">
  Use an RPC endpoint from the [RPCs list](/networks/rpcs). Some endpoints may be temporarily unavailable, if one fails, try another from the list.
</Callout>


## 3. Use the example contract

The `forge init` command automatically creates an example `Counter.sol` contract in `src/Counter.sol`. 


## 4. Use the example deploy script

The `forge init` command automatically creates an example deploy script in `script/Counter.s.sol`.


## 5. Manage keys securely

Use Foundry's encrypted keystore and add your private key when prompted:

```bash
cast wallet import eden-deployer --interactive
```

<Callout type="warning" emoji="🔐">
  Your private key will be used to sign transactions on the blockchain. Never share your private key with anyone, and always use secure key management practices. Never store private keys in plain text files like `.env`.
</Callout>

## 6. Deploy your contract

Deploy with:

```bash
forge script script/DeployCounter.s.sol:DeployCounter \
  --rpc-url eden \
  --chain-id 1234 \
  --broadcast \
  --account eden-deployer
```

<Callout type="info">
  The chain ID {EDEN_NETWORKS.testnet.chainId} is for Eden testnet. For mainnet, use chain ID {EDEN_NETWORKS.mainnet.chainId}. See the [RPCs page](/networks/rpcs) for current values.
</Callout>

If the RPC does not support [EIP-1559](https://eips.ethereum.org/EIPS/eip-1559), add `--legacy`.


## 7. Interact with the contract

You can interact with your contract using `cast` commands in your CLI:

```bash
# read state
cast call <DEPLOYED_ADDRESS> "number()(uint256)" --rpc-url eden

# send tx (increment count)
cast send <DEPLOYED_ADDRESS> "increment()" --rpc-url eden --account eden-deployer

# set a specific number
cast send <DEPLOYED_ADDRESS> "setNumber(uint256)" 42 --rpc-url eden --account eden-deployer
```

Or programmatically by creating custom scripts as in the following example:

```solidity
// SPDX-License-Identifier: UNLICENSED
pragma solidity ^0.8.13;

import {Script} from "forge-std/Script.sol";
import {console2} from "forge-std/console2.sol";
import {Counter} from "../src/Counter.sol";

contract InteractCounter is Script {
    // The deployed contract address
    address constant COUNTER_ADDRESS = 0x3f5bB2bd20147FbA8b275dEf2375aF04116f1cc5;
    
    Counter public counter;

    function setUp() public {
        counter = Counter(COUNTER_ADDRESS);
    }

    function run() public {
        vm.startBroadcast();

        console2.log("=== Counter Contract Interaction ===");
        console2.log("Contract Address:", address(counter));
        
        // Read current number
        uint256 currentNumber = counter.number();
        console2.log("Current number:", currentNumber);
        
        // Increment the counter
        console2.log("Incrementing counter...");
        counter.increment();
        
        // Read the new number
        uint256 newNumber = counter.number();
        console2.log("New number after increment:", newNumber);
        
        // Set a specific number
        uint256 targetNumber = 42;
        console2.log("Setting number to:", targetNumber);
        counter.setNumber(targetNumber);
        
        // Read the final number
        uint256 finalNumber = counter.number();
        console2.log("Final number:", finalNumber);
        
        console2.log("=== Interaction Complete ===");

        vm.stopBroadcast();
    }
}
```

## 9. Contract Verification

Contract verification is currently under development for Eden. The standard Foundry verification methods (Sourcify, Etherscan) do not yet support chain ID {EDEN_NETWORKS.testnet.chainId}.

For now, you can:
- View your deployed contracts on the [explorer]({EDEN_NETWORKS.testnet.explorer})
- Interact with contracts using `cast` commands or custom scripts
- Monitor the Eden testnet status for verification feature updates


## 10. Troubleshooting

### RPC Endpoint Issues
If you get "503 Service Unavailable" errors, try different RPC endpoints from the [RPC list](/networks/rpcs).

### Check Network Status
```bash
# Test RPC connectivity
curl -s http://rpc-evnode-edennet-1-testnet.binary.builders:8080 -X POST -H "Content-Type: application/json" -d '{"jsonrpc":"2.0","method":"eth_blockNumber","params":[],"id":1}'

# Check Eden testnet status
curl -s https://status-edennet-1-testnet.binary.builders:8443/status/edennet-1
```

### Common Issues
- **RPC timeouts**: Try different endpoints from the [RPC list](/networks/rpcs) or wait and retry
- **Gas estimation failures**: Ensure you have sufficient testnet TIA from the [faucet](/networks/faucet)
